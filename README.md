# 系統需求規格書

## 1. 文件目的
本文檔旨在定義一個基於 Java Swing 開發的「自動戰鬥地牢爬行遊戲」之進階功能需求。本文件將作為開發框架，除了核心戰鬥外，更引入了 Roguelike 元素、經濟決策與深度策略機制。

## 2. 遊戲核心流程 (Game Loop)
系統必須實現以下週而復始的進度循環：
1. **整備階段 (Shop & Prep Phase)**：玩家於商店招募英雄、合成單位、分配裝備。
2. **佈署階段 (Deployment Phase)**：調整陣型，觸發角色間的羈絆效果。
3. **戰鬥階段 (Combat Phase)**：執行自動戰鬥 AI，計算碰撞、技能與位移。
4. **結算階段 (Resolution Phase)**：發放金幣獎勵（利息、連勝/連敗加成）、更新地牢層數。

## 3. 策略佈局與單位深度 (Strategy & Depth)
### 3.1 網格化戰場 (Grid Map)
* **雙陣營對壘**：戰場劃分為 $8 \times 8$ 或 $6 \times 7$ 網格，玩家區域與敵方區域對等。
* **動態地形**：部分關卡產生「障礙物」（如岩石、岩漿），影響尋路與射程。

### 3.2 單位進化與羈絆 (Evolution & Synergies)
* **星級系統**：三個相同的 1 星單位自動合成 1 個 2 星單位（數值翻倍），最高 3 星。
* **職業與種族 (Tags)**：
    * 單位擁有多重標籤（如：戰士、法師、亡靈、精靈）。
    * **羈絆效果**：當場上部署不同種類的同標籤單位達到閾值（如 3/6/9）時，賦予全隊或特定單位的屬性加成（如：+20% 攻擊力、吸血效果）。

### 3.3 技能系統 (Ability System)
* **能量槽 (Mana Bar)**：單位透過攻擊或受擊獲取能量。
* **主動技能**：能量全滿時觸發，包含：
    * **範圍傷害 (AOE)**：對周圍或直線敵人造成傷害。
    * **控制技能 (CC)**：眩暈、冰凍或擊退敵人。
    * **增益/減益 (Buff/Debuff)**：治療隊友或削弱敵人護甲。

## 4. 自動化戰鬥邏輯 (Advanced AI Logic)

### 4.1 智能尋路與排隊
* **A* 尋路演算法**：單位須能避開障礙物與友方單位，尋找最短路徑。
* **攻擊偏好**：刺客單位優先跳躍至敵方後排；坦克單位優先挑釁距離最近的對手。

### 4.2 戰鬥屬性計算
* **護甲與抗性**：計算物理/魔法減傷公式。
* **暴擊與閃避**：隨機數判定攻擊是否產生爆發傷害或被完全規避。

## 5. 經濟與 Roguelike 系統 (Economy & Progression)
### 5.1 商店機制 (The Shop)
* **隨機刷新**：每回合隨機出現 5 個英雄供玩家購買。
* **升級人口**：玩家可花費金幣提升玩家等級，從而部署更多單位上場。
* **機率控制**：玩家等級越高，出現高稀有度（橙色、紫色）單位的機率越高。

### 5.2 裝備系統 (Itemization)
* **戰利品掉落**：擊敗特定層數的 Boss 後掉落裝備。
* **裝備賦予**：玩家可將裝備拖放到單位身上。裝備包含：
    * 增加攻擊速度、攻擊距離。
    * 物理/魔法穿透。
    * 復活甲或反傷甲。

## 6. 使用者介面需求 (UI Requirements)
### 6.1 視覺回饋 (Visual Feedback)
* **粒子效果**：受傷、施放技能、單位升星時的視覺特效。
* **羈絆面板**：左側顯示目前啟動的所有羈絆清單及進度（如：戰士 2/3）。

### 6.2 互動功能 (Interaction)
* **右鍵詳解**：右鍵點擊單位可開啟詳細面板，顯示當前 DPS（每秒傷害）與具體數值。
* **智能預測**：在佈署階段，顯示單位預計的攻擊路徑或影響範圍。

## 7. 資料與擴展性 (Architecture & Scalability)
* **JSON 驅動**：單位數值、技能參數、裝備屬性均儲存在 JSON 文件中，便於非開發者調整平衡性。
* **存檔系統**：記錄玩家當前關卡、金幣、剩餘血量與目前陣容，支援中斷重連。
* **多執行緒處理**：邏輯計算與 Swing 渲染分離，確保在高數量單位戰鬥時畫面不卡頓。

## 8. 成功判定與失敗條件
* **玩家血量 (Player HP)**：每場戰鬥失敗後，根據場上剩餘敵方單位數量與等級扣除玩家血量。
* **通關目標**：擊敗第 50 層的最終 Boss 為勝利；玩家血量歸零則遊戲結束並進入結算畫面。
